C++：通用算法

大多数算法都定义在algorithm头文件中；某些跟数字有关的算法定义在numeric头文件中。

只读算法：

函数 含义
find(begin, end, val) 在迭代器$[begin, end)$范围内找到值为val的元素，返回指向它的迭代器。
find_if(begin, end, predicate) 在迭代器$[begin,end)$表示的范围内找到第一个满足断言的元素，返回指向它的迭代器
accumulate(begin, end,val) 计算从val+*begin + … + *(end - 1)
equal(begin1, end1, begin2) 判断两个序列是否含有相同的值。函数假定第二个序列的长度至少和第一个序列相等
count_if(begin, end, predicate) 计算范围$[begin, end)$范围含有的元素中满足断言的元素数量。

lambda 表达式

可以将任何可调用的对象传递给一个算法。一个可调用的对象指的是可以在其上使用函数调用操作符()的对象。lambda表达式描述了一个可调用的单元：

capture list表示lambda表达式的捕获局部变量列表，这些变量在表达式内部可以使用；lambda表达式可以使用定义在定义表达式函数外部的变量。

可以省略参数列表和返回列表，但是不能省略函数体。

lambda表达式变量捕获：

当定义一个lambda表达式时，编译器会为表达式自动生成一个匿名类。生成的匿名类中含有捕获列表中的指定的变量作为类的成员。

两种捕获形式：值捕获和引用捕获：
###值捕获：

默认捕获为值捕获，捕获是在lambda表达式被创建时进行，因此对变量就进行任何后续的修改都无法影响到lambda表达式，且lambda表达式对变量的修改无法在外部观察到。

引用捕获：

引用捕获必须保证变量在表达式被执行时存在。因为表达式在函数内部被声明，函数退出时局部变量将不存在。

使编译器自动捕获变量：方括号中只含有=号表示编译器自动值捕获局部变量，只含有&号表示编译器自动引用捕获局部变量。

符号 含义
[] 无需要捕获的变量
[=] 自动按值捕获变量
[&] 自动按引用捕获变量
[&,identifier_list] 除了identifier_list中列出的变量用值捕获之外，其他的变量用引用捕获
[=,reference_list] 除了reverence_list中列出的变量用引用捕获之外，其他的变量用值捕获

可修改的lambda表达式（Mutbale Lambdas）：

对于值捕获的变量，如果需要在lambda表达式内部修改他的值，需要添加mutable修饰符：添加修饰符后参数列表不可省略。如果不加mutable修饰符，值捕获的变量不能在内部被修改。

lambda表达式的返回类型：

lambda表达式中如果只含有一个return语句，那么lambda表达式的返回类型由该return语句决定，否则需要显示指明lambda表达式的返回类型。如果没有return语句编译器认为lambda表达式的返回类型是void。

迭代器

除了容器提供的迭代器之外，iterator提供了多种类型的迭代器，包括：

  • 插入迭代器：绑定到容器中，可以用于向容器中插入元素；
  • 流迭代器：绑定到绑定待输入输出流中，可以用来迭代相应的IO流；
  • 反向迭代器：前向迭代；
  • 移动迭代器：特殊用途的迭代器，移动而不是复制元素。

插入迭代器：

向插入迭代器赋值时，迭代器通过调用容器的函数将元素插入到容器的指定位置中。

  • back_inserter()创建使用push_back()函数插入元素的迭代器；
  • front_inserter()创建使用push_front()函数插入元素的迭代器；
  • inserter(c iter)创建使用insert()函数插入元素的迭代器。
操作 含义
it = t 向it绑定的容器中插入元素t；
*it，++it，it++ 无意义，返回it。

  • ostream_iterator：用<<操作符向输出流中写数据。

反向迭代器

反向迭代器中的++操作符意味着向前移动，--意味着向后移动。因为forward_list和流不支持后向移动，所以不能为这些类型创建反向迭代器。从已经有的迭代器创建反向迭代器要求该迭代器支持++和--操作，可以通过反向迭代器的base方法获取相应的正向迭代器。需要注意的是正向迭代器指向的是反向迭代器的下一个元素：

通用算法的结构

通用算法根据其行为，对使用的迭代器进行了如下的分类：

分类 支持的操作
输入迭代器 读，single-pass，只需要增加，即==和!=，++，*，->
输出迭代器 写，single-pass，只需要增加，即++，*
前向迭代器 读和写，multi-pass，增，即==和!=，++，*，->，同时可以保存之前状态的迭代器；
双向迭代器 读和写，multi-pass，增和减；==和!=，++，*，->，--；
随机访问迭代器 读和写，multi-pass，所有的迭代器算数操作，即==和!=，++，*，->，<，<=，>，>=，+，+=，-，-=，两个迭代器的减法，[]（iter[n]与*(iter + n)含义相同）

容器专用算法

由于list和forward_list的特殊性，用在这些容器上的算法被定义为了成员函数。这些成员函数返回void类型

函数 含义
lst.merge(lst2) 将lst合并进lst2，lst和lst2须排序。用<比较lst和lst2的元素；合并完后，lst2为空。
lst.merge(lst2, comp) 将lst合并进lst2，lst和lst2须排序。用cpmp比较lst和lst2的元素；合并完后，lst2为空。
lst.remove(val) 调用erase函数删除所有==val的元素；
lst.remove_if(pred) 调用erase函数删除所有满足断言pred的元素；
lst.reverse() 将lst中的顺序反过来；
lst.sort() 用<排序lst中的元素；
lst.sort(comp) 用comp排序lst中的元素；
lst.unique() 用==号去连续相同的元素；
lst.unique(pred) 用pred去连续相同的元素；

lst.splice(args)和flst.splice_after(args)：

参数 含义
(p, lst2) 将lst2中的元素移动进lst中迭代器p指向的元素之后(lst)或之前(flst)
(p, lst2, p2) 将lst2中的p指向元素开始(lst)或之后(flst)的元素移动进lst中迭代器p指向的元素之后(lst)或之前(flst)，lst2可以是lst/flst。
(p, lst2, b, e) 将lst2中范围$[begin, end)$的元素移动进lst中迭代器p指向的元素之后(lst)或之前(flst)，lst2可以是lst/flst但是p不能包含在$[begin, end)$中。
