vim 常用命令

指令模式( command line)

  • 移动光标方向

    上下左右移动：

    • h或者左方向键左移，j或者上方向键上移，k或者下方向键下移，l或者右方向键右移动；想要在一个方向上移动多行的话，是数字加上h，j，k，l。
    • - 和 + 代表移动到非空白字符的上一列和下一列；
    • 0和$代表移动到这一列的第一个和最后一个字符；
    • n + <space> 代表向右移动n个字符；
    • n + <enter> 表示向下移动n行；
    • L表示移动到屏幕显示的最后一列的第一个字符上面；G表示移动到最后一列，nG表示移动到这个文件的第n列；gg表示移动到第一列，相当于1G；

C++：通用算法

大多数算法都定义在algorithm头文件中；某些跟数字有关的算法定义在numeric头文件中。

只读算法：

函数 含义
find(begin, end, val) 在迭代器$[begin, end)$范围内找到值为val的元素，返回指向它的迭代器。
find_if(begin, end, predicate) 在迭代器$[begin,end)$表示的范围内找到第一个满足断言的元素，返回指向它的迭代器

count_if(begin, end, predicate) 计算范围$[begin, end)$范围含有的元素中满足断言的元素数量。

顺序容器概览

容器 头文件 含义
vector vector 动态大小容器，且提供快速随机访问。
deque deque 双端队列实现
list list 双向链表实现

类的范围

因为一个类用花括号括起来了，所以这个类也指定了一个scope。

定义在类的定义之外的成员函数：

定义在类定义之外的成员函数，因为通过::指定了该函数是某一个类的成员函数，所以在函数名中::之前的部分，如返回类型是不算在类的范围里面的，因此如果返回类型使用了类中定义的类型，那么需要用::显示指定；而在函数名中::之后的部分，因编译器已经知道该函数是某一类的成员函数，因此不需要加::指定。

定义在类的定义之内的成员函数：

类的编译过程如下：

  1. 编译类中所有成员的声明，在类中定义的类只编译其函数头部，即返回值、函数名、参数列表部分，函数体不会编译；
  2. 编译类中定义的成员函数的函数体。

这样的结果就是函数体重可以访问类中的所有成员，但是函数定义部分使用的类型必须在函数声明之前定义。
